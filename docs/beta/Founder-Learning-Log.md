# Founder Learning Log

**Purpose:** the single place real signal from Stage A private beta customers gets recorded, so it can accumulate into something the roadmap actually learns from — not scattered across memory, Slack messages, and founders' individual impressions. This log is the qualitative record [Beta-Success-Metrics.md](Beta-Success-Metrics.md)'s qualitative criteria and [Version-1.0-Roadmap.md](../plans/Version-1.0-Roadmap.md)'s stage-gate decisions should draw from.

**How to use this document:**
- Create one entry per customer per checkpoint — onboarding, first recommendation, week one, month one (see [Customer-Interview-Guide.md](Customer-Interview-Guide.md)) — plus any ad hoc conversation worth recording.
- Fill in an entry within 24 hours of the conversation, while it's fresh. A memory of "they seemed happy" a week later is not useful data.
- Don't wait for a "big" learning to log an entry. A five-minute observation during onboarding is worth recording — small, frequent entries beat sparse, dramatic ones.
- Log entries newest-first, so the most recent signal is always at the top when this document is reopened during the daily/weekly review cadence in [Private-Beta-Execution.md](../plans/Private-Beta-Execution.md) §3 and §5.
- Be honest here even when — especially when — something didn't go well. This log is only useful if it's a true record, not a highlight reel.
